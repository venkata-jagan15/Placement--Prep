std::vector #include

Under the hood: A Dynamic Array storing elements contiguously in memory. Because memory is contiguous, CPU Cache utilization is phenomenal, making vector the absolute fastest container in C++ for general traversal.

🎯 General Use Case

Use for almost everything by default. Best when you need fast O(1) access by index and only need to add/remove elements at the END.

💻 Initialization Syntax

vector<DataType> vectorName;
vector<DataType> vectorName(size, initialValue);
vector<vector<DataType>> matrixName(rows, vector<DataType>(cols, initialValue));

⚠ Watch Out

Do NOT use if you need to frequently insert or erase elements at the FRONT or MIDDLE. This takes O(N) time because all subsequent elements must be physically shifted in memory.

Detailed Methods & Examples

push_back(val) Amortized O(1)

Description: Adds element to the end. Creates a copy of the object.

🎯 Deep Use Case: Standard array building. Extremely common when you don't know the final size of the array ahead of time.
Examples
// Ex 1: Basic insertion\nvector v;\nv.push_back(10);\nv.push_back(20);
// Ex 2: String vector\nvector words;\nwords.push_back("Hello");
// Ex 3: Building a Graph Adjacency List\nvector> adj(5);\nadj[0].push_back(1); // Directed Edge from node 0 to 1
emplace_back(args...) Amortized O(1)

Description: Constructs the element in-place at the end. Faster than push_back for pairs/objects.

🎯 Deep Use Case: Optimization. Whenever you are pushing a Pair, Struct, or Object, use emplace_back to avoid creating a temporary object and copying it.
Examples
// Ex 1: Emplacing pairs directly (No {} needed)\nvector> v;\nv.emplace_back(1, "apple"); // Instantly created inside vector
// Ex 2: Emplacing a custom struct\nstruct Node { int x, y; };\nvector nodes;\nnodes.emplace_back(10, 20);
pop_back() O(1)

Description: Removes the last element. Does not return it.

🎯 Deep Use Case: Crucial for Backtracking (DFS). When exploring a path, you push the node. When returning from the recursive call, you pop it off to try another path.
Examples
// Ex 1: Standard pop\nvector v = {1, 2, 3};\nv.pop_back(); // {1, 2}
// Ex 2: The Backtracking Pattern\npath.push_back(node);\ndfs(node, path); // Recursive call\npath.pop_back(); // Undo the choice (Backtrack)
v[i] or v.at(i) O(1)

Description: Access element. .at(i) throws exception if out of bounds. v[i] is unchecked and slightly faster.

🎯 Deep Use Case: Accessing DP tables, reading specific indices. In competitive programming, always use v[i] for speed.
Examples
// Ex 1: Fast array access\nint x = v[0];
// Ex 2: Safe access (good for debugging)\ntry { cout << v.at(100); }\ncatch (const std::out_of_range& e) { cout << "Out of Bounds!"; }
front() / back() O(1)

Description: Returns reference to first/last element.

🎯 Deep Use Case: Getting the most recently added element (back) or the oldest (front) without needing to calculate the index.
Examples
// Ex 1: Getting ends\nvector v = {1, 2, 3};\ncout << v.front(); // 1\ncout << v.back(); // 3
// Ex 2: Modifying ends\nv.back() = 99; // {1, 2, 99}
size() / empty() O(1)

Description: Returns number of elements / true if size is 0.

🎯 Deep Use Case: Looping bounds, or checking if there's still data left to process.
Examples
// Ex 1: Standard loop\nfor(int i=0; i
// Ex 2: While loop safety\nwhile(!v.empty()) {\n    cout << v.back();\n    v.pop_back();\n}
resize(n, val) O(N)

Description: Resizes the vector to contain n elements. New elements are initialized to val.

🎯 Deep Use Case: Pre-allocating space for DP tables (Dynamic Programming) when the dimensions are known.
Examples
// Ex 1: 1D DP table\nvector dp;\ndp.resize(100, -1); // 100 elements initialized to -1
// Ex 2: Resizing a 2D matrix\nvector> matrix;\nmatrix.resize(5, vector(5, 0)); // 5x5 matrix of zeros
clear() O(N)

Description: Destroys all elements (size becomes 0). Capacity usually remains unchanged.

🎯 Deep Use Case: Resetting a global vector between multiple test cases without reallocating memory.
Examples
// Ex 1: Clearing state for next test case\nvector visited;\nvoid solve() {\n    visited.clear();\n    // ... run algorithm\n}
insert(iterator, val) O(N)

Description: Inserts element before iterator. Shifts everything to the right.

🎯 Deep Use Case: Use sparingly. Only used if you absolutely must insert in the middle of a small array.
Examples
// Ex 1: Insert at beginning\nvector v = {2, 3};\nv.insert(v.begin(), 1); // {1, 2, 3}
// Ex 2: Insert multiple copies\nv.insert(v.begin() + 1, 3, 99); // {1, 99, 99, 99, 2, 3}
erase(iterator) O(N)

Description: Removes element. Shifts everything left.

🎯 Deep Use Case: Removing a specific element from the middle of an array when order must be preserved.
Examples
// Ex 1: Erase a single element\nvector v = {10, 20, 30, 40};\nv.erase(v.begin() + 1); // Removes 20 -> {10, 30, 40}
// Ex 2: Erase a range\nv.erase(v.begin(), v.begin() + 2); // Removes first 2 elements

std::list #include

Under the hood: A Doubly Linked List. Elements are scattered in memory, connected via pointers. There is NO contiguous memory, which means you CANNOT access elements via index (e.g., l[5] is a compile error).

🎯 General Use Case

Use when you have massive amounts of insertions and deletions in the MIDDLE or FRONT of the container, provided you already have an iterator pointing to that location.

💻 Initialization Syntax

list<DataType> listName;
list<DataType> listName = {val1, val2, val3};

⚠ Watch Out

Do NOT use if you need random access. Poor CPU cache locality compared to vector. High memory overhead per element.

Detailed Methods & Examples

push_front(val) / pop_front() O(1)

Description: Adds or removes element to/from the beginning instantly.

🎯 Deep Use Case: Building a custom Deque, or prepending items to a linked list.
Examples
// Ex 1: Push front\nlist l;\nl.push_front(10);\nl.push_front(20); // {20, 10}
// Ex 2: Pop front\nl.pop_front(); // {10}
push_back(val) / pop_back() O(1)

Description: Adds or removes element to/from the end instantly.

🎯 Deep Use Case: Standard queue-like behavior.
Examples
// Ex 1: Push back\nlist l;\nl.push_back(10);\nl.push_back(20); // {10, 20}
insert(iterator, val) O(1)

Description: Inserts before iterator instantly (unlike vector, which is O(N)).

🎯 Deep Use Case: The main reason to use a list. If you know exactly where a node belongs, insertion costs zero shifting.
Examples
// Ex 1: Insert in middle\nlist l = {10, 30};\nauto it = l.begin();\nit++; // Point to 30\nl.insert(it, 20); // {10, 20, 30} in O(1) time
erase(iterator) O(1)

Description: Removes element instantly.

🎯 Deep Use Case: Removing an item without shifting arrays. Crucial for LRU cache eviction.
Examples
// Ex 1: Erase second element\nauto it = ++l.begin();\nl.erase(it); // O(1) removal
splice(iterator, list2) O(1)

Description: Transfers all elements from list2 into the calling list without reallocating any memory. It just re-assigns the pointers.

🎯 Deep Use Case: Moving nodes around instantly. This is the secret to building a highly optimized O(1) LRU Cache (moving accessed item to the front of the MRU list).
Examples
// Ex 1: Moving a specific node to the front\nlist lru = {3, 2, 1};\nauto it = ++lru.begin(); // points to 2\nlru.splice(lru.begin(), lru, it); // {2, 3, 1}
sort() O(N log N)

Description: Special list sort (std::sort requires random access, so you MUST use list's built-in .sort()).

🎯 Deep Use Case: Sorting a linked list.
Examples
// Ex 1: Sorting\nlist l = {5, 2, 8};\nl.sort(); // {2, 5, 8}
// Ex 2: Descending sort\nl.sort(greater()); // {8, 5, 2}

std::deque #include

Under the hood: A Double-Ended Queue. Think of it as an array of fixed-size arrays. It allows fast O(1) insertion/deletion at BOTH ends, and also provides random access via index `dq[i]`.

🎯 General Use Case

Use exactly like a Vector, but when you specifically need to add or remove elements from the FRONT frequently.

💻 Initialization Syntax

deque<DataType> dq;
deque<DataType> dq(size, initialValue);

⚠ Watch Out

Insertion in the middle still takes O(N). Memory overhead is slightly higher than vector.

Detailed Methods & Examples

push_front(val) / pop_front() O(1)

Description: Adds/Removes element to/from the front.

🎯 Deep Use Case: Sliding Window Maximum problem. You need to maintain a monotonic decreasing queue and pop elements from the front that fall out of the sliding window.
Examples
// Ex 1: Deque front operations\ndeque dq;\ndq.push_front(1);\ndq.push_front(2); // {2, 1}
// Ex 2: Sliding Window Pattern\nif(!dq.empty() && dq.front() == outOfWindowElement) {\n    dq.pop_front();\n}
push_back(val) / pop_back() O(1)

Description: Adds/Removes element to/from the back.

🎯 Deep Use Case: Also Sliding Window Maximum: popping elements from the back that are strictly less than the new element being added.
Examples
// Ex 1: Monotonic Queue maintenance\nwhile(!dq.empty() && dq.back() < currentVal) {\n    dq.pop_back();\n}\ndq.push_back(currentVal);
dq[i] O(1)

Description: Random access by index. (Unlike std::list)

🎯 Deep Use Case: When you need queue capabilities but still want to inspect the N-th element.
Examples
// Ex 1: Index access\ndeque dq = {10, 20, 30};\ncout << dq[1]; // Prints 20

std::set & std::multiset #include

Under the hood: Implemented as a Red-Black Tree (Self-Balancing Binary Search Tree). Elements are ALWAYS strictly sorted. `set` stores only UNIQUE elements. `multiset` allows DUPLICATE elements.

🎯 General Use Case

Use when you need to store elements and constantly query the minimum, maximum, or closest values dynamically as elements are added or removed.

💻 Initialization Syntax

set<DataType> setName;
multiset<DataType> multisetName;
set<DataType, greater<DataType>> descendingSet;

⚠ Watch Out

Do NOT use if you just need to check if an element exists quickly and don't care about sorting. Use `unordered_set` instead (O(1) vs O(log N)).

Detailed Methods & Examples

insert(val) O(log N)

Description: Inserts element in its sorted position. If set, ignores duplicates.

🎯 Deep Use Case: Dynamically keeping a running stream of numbers perfectly sorted.
Examples
// Ex 1: Auto sorting\nset s;\ns.insert(50);\ns.insert(20); // Internally sorts to {20, 50}
// Ex 2: Multiset duplicates\nmultiset ms;\nms.insert(10); ms.insert(10); // {10, 10}
erase(val) O(log N)

Description: Removes the element. WARNING: In multiset, passing a value removes ALL instances of that value.

🎯 Deep Use Case: Standard removal from a binary search tree.
Examples
// Ex 1: Standard erase\nset s = {1, 2, 3};\ns.erase(2); // {1, 3}
// Ex 2: Multiset Trap\nmultiset ms = {1, 1, 1, 2};\nms.erase(1); // ms is now {2}. ALL 1s are gone!
erase(iterator) Amortized O(1)

Description: Removes exactly one element pointed to by the iterator.

🎯 Deep Use Case: Crucial for multiset! When sliding a window, you want to remove exactly ONE instance of the outgoing number.
Examples
// Ex 1: Multiset Safe Erase (ONE instance)\nmultiset ms = {1, 1, 1, 2};\nms.erase(ms.find(1)); // ms is now {1, 1, 2}
find(val) / count(val) O(log N)

Description: find() returns an iterator. count() returns the number of occurrences.

🎯 Deep Use Case: Checking existence in a sorted collection. count() is great for finding frequency in a multiset.
Examples
// Ex 1: Checking existence\nif(s.find(5) != s.end()) cout << "Found!";
// Ex 2: Frequency check\nmultiset ms = {5, 5, 5};\ncout << ms.count(5); // 3
lower_bound(val) O(log N)

Description: Returns iterator to FIRST element that is >= val.

🎯 Deep Use Case: Finding the smallest available box that can fit a package of size `val`.
Examples
// Ex 1: Lower bound\nset s = {10, 20, 30};\nauto it = s.lower_bound(15);\ncout << *it; // Prints 20
// Ex 2: Exact match\nauto it2 = s.lower_bound(20);\ncout << *it2; // Prints 20
upper_bound(val) O(log N)

Description: Returns iterator to FIRST element that is strictly > val.

🎯 Deep Use Case: Finding the next strictly larger element in a sorted tree.
Examples
// Ex 1: Upper bound\nset s = {10, 20, 30};\nauto it = s.upper_bound(20);\ncout << *it; // Prints 30
begin() / end() / rbegin() / rend() O(1)

Description: Iterators for the tree. *begin() gives the smallest element. *rbegin() gives the largest.

🎯 Deep Use Case: Instantly retrieving the global minimum or maximum from the dynamically sorted tree.
Examples
// Ex 1: Get Minimum element\ncout << *s.begin();
// Ex 2: Get Maximum element\ncout << *s.rbegin();

std::map & std::multimap #include

Under the hood: Implemented as a Red-Black Tree. Stores Key-Value pairs. It guarantees that the Keys are strictly sorted. All operations are O(log N).

🎯 General Use Case

Use when you need a dictionary, but you explicitly need to iterate over the keys in sorted order, or you need to find the `lower_bound` of a Key.

💻 Initialization Syntax

map<KeyType, ValueType> mapName;
map<KeyType, ValueType, greater<KeyType>> descendingMap;

⚠ Watch Out

Do NOT use just for counting frequencies. `unordered_map` is significantly faster.

Detailed Methods & Examples

m[key] = val O(log N)

Description: Inserts or updates value for key.

🎯 Deep Use Case: Mapping a sorted timeline of keys to specific values.
Examples
// Ex 1: Basic insertion\nmap m;\nm[20] = "Apple";\nm[10] = "Banana"; // Keys are sorted 10 -> 20
insert({key, val}) O(log N)

Description: Inserts pair. Fails silently if key already exists.

🎯 Deep Use Case: When you want to ensure you don't accidentally overwrite an existing value for a key.
Examples
// Ex 1: Safe insert\nm.insert({10, "Cherry"}); // Does nothing if 10 exists
erase(key) O(log N)

Description: Removes key and its value.

🎯 Deep Use Case: Removing an entry from the sorted dictionary.
Examples
// Ex 1: Erasing by key\nm.erase(10);
lower_bound(key) O(log N)

Description: Finds first Pair with Key >= key.

🎯 Deep Use Case: Time-based key-value store. E.g., 'What was the value of X at timestamp T, or the closest timestamp after T?'
Examples
// Ex 1: Map lower bound\nmap m = {{10, "A"}, {30, "B"}};\nauto it = m.lower_bound(20);\ncout << it->first << "-" << it->second; // 30-B

std::unordered_map & unordered_set #include /

Under the hood: Implemented using a Hash Table. Keys are hashed to buckets. Zero ordering. Average time complexity is purely O(1).

🎯 General Use Case

The ultimate containers for frequency counting, caching, Memoization, and simple existence checks.

💻 Initialization Syntax

unordered_map<KeyType, ValueType> mapName;
unordered_set<DataType> setName;

Detailed Methods & Examples

m[key] = val Avg O(1)

Description: Accesses or inserts a value. WARNING: Accessing a non-existent key inserts it with a default value (like 0).

🎯 Deep Use Case: Counting frequencies of elements in an array, or caching results in Dynamic Programming (Memoization).
Examples
// Ex 1: Frequency Array\nunordered_map freq;\nvector nums = {1, 2, 2, 3};\nfor(int num : nums) freq[num]++;
// Ex 2: DP Memoization\nunordered_map memo;\nint fib(int n) {\n  if(memo.count(n)) return memo[n];\n  return memo[n] = fib(n-1) + fib(n-2);\n}
find(key) Avg O(1)

Description: Returns an iterator to the pair. Returns m.end() if missing. Does NOT create a new key.

🎯 Deep Use Case: Checking if you have seen a number before, like in the classic 'Two Sum' problem.
Examples
// Ex 1: Two Sum logic using find\nunordered_map seen; // value -> index\nfor(int i=0; i
count(key) Avg O(1)

Description: Returns 1 if exists, 0 otherwise.

🎯 Deep Use Case: A cleaner syntax than find() when you just need a boolean existence check.
Examples
// Ex 1: Clean existence check\nif(seen.count(needed)) { return true; }
// Ex 2: unordered_set check\nunordered_set mySet = {1, 5, 9};\nif(mySet.count(5)) { cout << "Exists!"; }
erase(key) Avg O(1)

Description: Removes the key-value pair from the hash table.

🎯 Deep Use Case: Removing seen elements, or backtracking state.
Examples
// Ex 1: Removing a cached value\nmemo.erase(n);

std::stack #include

Under the hood: A Container Adaptor restricting access to LIFO (Last In, First Out).

🎯 General Use Case

Use for Monotonic Stacks (Next Greater Element), Valid Parentheses, Iterative DFS.

💻 Initialization Syntax

stack<DataType> stackName;
stack<pair<int, int>> stackOfPairs;

Detailed Methods & Examples

push(val) / pop() / top() O(1)

Description: Push adds to top. Pop removes top (returns void). Top gets value at top.

🎯 Deep Use Case: Validating expressions or running a Monotonic Stack to find the Next Greater Element.
Examples
// Ex 1: Valid Parentheses\nstack st;\nfor(char c : s) {\n  if(c == '(') st.push(c);\n  else if(!st.empty() && st.top() == '(') st.pop();\n}
// Ex 2: Monotonic Decreasing Stack\nstack st;\nwhile(!st.empty() && st.top() < current) {\n    cout << "Next greater is " << current;\n    st.pop();\n}\nst.push(current);
empty() / size() O(1)

Description: Checks if size is 0 / returns number of elements.

🎯 Deep Use Case: ALWAYS use empty() before calling top() or pop(), or your program will crash.
Examples
// Ex 1: Safe pop\nif(!st.empty()) { st.pop(); }

std::queue #include

Under the hood: A Container Adaptor following FIFO (First In, First Out).

🎯 General Use Case

The core data structure for Breadth First Search (BFS) in trees and graphs.

💻 Initialization Syntax

queue<DataType> queueName;
queue<TreeNode*> treeQueue;

Detailed Methods & Examples

push(val) / pop() / front() O(1)

Description: Push adds to back. Pop removes from front. Front gets first element.

🎯 Deep Use Case: Level order traversal of a binary tree or graph.
Examples
// Ex 1: BFS using Queue\nqueue q;\nq.push(root);\nwhile(!q.empty()) {\n  TreeNode* node = q.front(); q.pop();\n  if(node->left) q.push(node->left);\n  if(node->right) q.push(node->right);\n}

std::priority_queue #include

Under the hood: A Binary Heap. It ensures that the 'largest' (or 'smallest') element is always at the top. Built on top of a vector.

🎯 General Use Case

Dijkstra's Algorithm, Top K elements, finding running median.

💻 Initialization Syntax

// Max Heap (Largest at top)
priority_queue<int> maxHeap;

// Min Heap (Smallest at top - MEMORIZE THIS!)
priority_queue<int, vector<int>, greater<int>> minHeap;

// Min Heap for Pairs (Sorts by first element ascending)
priority_queue<pair<int,int>, vector<pair<int,int>>, greater<pair<int,int>>> pq;

Detailed Methods & Examples

push(val) / pop() O(log N)

Description: Inserts element and sifts up/down to maintain heap property.

🎯 Deep Use Case: Continually feeding data into a Top K elements tracker.
Examples
// Ex 1: Maintaining Top 3 elements (using Min Heap)\npriority_queue, greater> minHeap;\nminHeap.push(val);\nif(minHeap.size() > 3) minHeap.pop(); // Toss out the smallest
top() O(1)

Description: Returns reference to the highest priority element.

🎯 Deep Use Case: Extracting the node with the minimum distance in Dijkstra's.
Examples
// Ex 1: Min Heap syntax\npriority_queue, greater> minHeap;\ncout << minHeap.top();
// Ex 2: Using the pair\nauto [dist, node] = pq.top(); pq.pop();

make_pair(a, b) / {a, b} O(1)

Description: Creates a pair.

🎯 Deep Use Case: Bundling a grid coordinate (row, col) to put inside a queue for BFS.
Examples
// Ex 1: Pair creation\npair p = {1, "apple"};
// Ex 2: Graph BFS coordinate\nqueue> q;\nq.push({0, 0});
p.first / p.second O(1)

Description: Direct access to pair members.

🎯 Deep Use Case: Extracting the bundled data.
Examples
// Ex 1: Extraction\ncout << p.first << " - " << p.second;
get(tuple) O(1)

Description: Access tuple elements (for > 2 values).

🎯 Deep Use Case: When you need to store (distance, row, col) in a priority queue.
Examples
// Ex 1: Tuple usage\ntuple t = {1, 2, "hello"};\ncout << get<2>(t); // "hello"

Algorithms #include /

Under the hood: Global functions operating on iterator ranges. They save you from writing standard loops.

🎯 General Use Case

Sorting, searching, and manipulating vectors efficiently.

💻 Initialization Syntax

// No specific initialization, these are free functions.\n// Just ensure you include <algorithm> and <numeric>

Detailed Methods & Examples

sort(begin, end, [comparator]) O(N log N)

Description: Sorts the range using IntroSort.

🎯 Deep Use Case: Preparing an array for Binary Search, or grouping anagrams.
Examples
// Ex 1: Ascending\nvector v = {3, 1, 4};\nsort(v.begin(), v.end());
// Ex 2: Descending\nsort(v.begin(), v.end(), greater());
// Ex 3: Custom Lambda Comparator\nsort(v.begin(), v.end(), [](pair& a, pair& b) {\n    return a.second < b.second; // sort by 2nd element\n});
binary_search(begin, end, val) O(log N)

Description: Returns true if value exists. Array MUST be sorted.

🎯 Deep Use Case: Quick existence check in a pre-sorted dataset.
Examples
// Ex 1: Basic usage\nbool exists = binary_search(v.begin(), v.end(), 5);
lower_bound / upper_bound O(log N)

Description: Finds first element >= val (lower) or strictly > val (upper) in a sorted array.

🎯 Deep Use Case: Finding insertion points or finding the closest element.
Examples
// Ex 1: Getting iterator\nauto it = lower_bound(v.begin(), v.end(), 3);
// Ex 2: Getting exact INDEX\nint index = lower_bound(v.begin(), v.end(), 3) - v.begin();
reverse(begin, end) O(N)

Description: Reverses the range in-place.

🎯 Deep Use Case: Reversing a string or vector without manually swapping.
Examples
// Ex 1: Reverse string\nstring s = "hello";\nreverse(s.begin(), s.end()); // "olleh"
accumulate(begin, end, init) O(N)

Description: Sums the elements in the range.

🎯 Deep Use Case: Quick summation without a for-loop.
Examples
// Ex 1: Standard sum\nint sum = accumulate(v.begin(), v.end(), 0);
// Ex 2: Prevent Overflow with Long Long!\nlong long hugeSum = accumulate(v.begin(), v.end(), 0LL);
next_permutation(begin, end) O(N)

Description: Rearranges the array to the next lexicographical permutation. Returns false if it was the last one.

🎯 Deep Use Case: Generating all permutations of a string or array.
Examples
// Ex 1: Generate all permutations\nvector v = {1, 2, 3};\ndo {\n    // process v\n} while(next_permutation(v.begin(), v.end()));
max_element / min_element O(N)

Description: Returns iterator to the max/min element.

🎯 Deep Use Case: Finding extremums instantly.
Examples
// Ex 1: Get Max Value\nint m = *max_element(v.begin(), v.end());
// Ex 2: Get Max Index\nint idx = max_element(v.begin(), v.end()) - v.begin();
