C++ Fundamentals & Memory

1. Compilation & Execution Theory

How C++ Works

Unlike Python (interpreted) or Java (bytecode/JVM), C++ code is compiled directly to machine code. This makes it incredibly fast, which is why it's heavily used in Competitive Programming and high-performance systems.

Pipeline: Preprocessing (expands #includes and macros) -> Compilation (converts to Assembly) -> Assembly (converts to object files .o/.obj) -> Linking (links libraries and object files to create an executable .exe/.out).

2. Memory Management: Stack vs. Heap

Understanding memory is the most critical part of C++ interviews.

  • Stack Memory: Used for local variables and function calls. Fast allocation, limited size. Managed automatically (when variables go out of scope, they are popped).
  • Heap Memory: Used for dynamic allocation. Vast size, slower allocation. You must manage it manually using new and delete. Failure to do so causes memory leaks.
// Stack allocation
int a = 10; 
int arr[100]; // Size must be known at compile time

// Heap allocation (Dynamic)
int* ptr = new int(10); // Allocates single int on heap
int* dynArr = new int[n]; // Size 'n' can be decided at runtime

delete ptr; // Prevent memory leak
delete[] dynArr; // Use delete[] for arrays

3. Pointers vs. References (The Deep Use Case)

Pointers: A variable that stores the memory address of another variable.
References: An alias to an existing variable. It cannot be NULL and cannot be reassigned once initialized.

Use Case in DSA: Passing by Reference

When you pass a large std::vector to a recursive function (like in Backtracking or DFS), if you pass by value, C++ creates a massive copy of the vector every single time. This turns an O(N) algorithm into O(N^2) and causes Time Limit Exceeded (TLE).

Solution: ALWAYS pass large structures by reference &.

// WRONG for DSA (Pass by value - slow, makes copy)
void dfs(vector path, int node) { ... }

// CORRECT for DSA (Pass by reference - fast, no copy)
// Use 'const' if you don't intend to modify it to prevent bugs.
void dfs(const vector& path, int node) { ... }

C++ Advanced Concepts (OOP & Modern C++)

1. Object-Oriented Theory (The 4 Pillars)

  • Encapsulation: Binding data and functions into a single unit (Class). Hiding internal states via private access modifiers to prevent unauthorized changes.
  • Abstraction: Exposing only necessary interfaces and hiding implementation details. Achieved using abstract classes and interfaces (classes with pure virtual functions).
  • Inheritance: Creating a new class from an existing class to promote code reusability.
  • Polymorphism: "Many forms". Compile-time (Method Overloading, Operator Overloading) and Run-time (Method Overriding using virtual keywords).

Interview Question: What is a Virtual Function?

A virtual function is a member function declared in a base class and redefined (overridden) by a derived class. It ensures that the correct function is called for an object, regardless of the type of reference (or pointer) used for function call. This is resolved via a V-Table (Virtual Table) at runtime (Dynamic Dispatch).

class Base {
public:
    virtual void show() { cout << "Base"; } // Virtual keyword
    virtual ~Base() {} // Always make base destructors virtual!
};

class Derived : public Base {
public:
    void show() override { cout << "Derived"; } 
};

int main() {
    Base* b = new Derived();
    b->show(); // Output: "Derived" (Because of 'virtual')
    delete b;
}

1. Sliding Window

▼

Deep Theory & Concept

Instead of using nested loops to calculate sums of all possible subarrays (O(N^2)), we maintain a 'window' defined by a left and right pointer. As we expand the right pointer, we add to our running state. When a condition breaks, we shrink the left pointer. This visits every element at most twice (once by right, once by left), yielding O(N) time.

Common Variants

1. Fixed Window (Find max sum of subarray of size K).
2. Variable Window (Find longest subarray with sum < K).
3. String Window (Longest substring with K distinct characters).

How to Identify: Linear data structures (arrays, strings). Keywords: 'longest', 'shortest', 'maximum', 'minimum' subarray/substring. 'Contiguous sequence'.

Approach to Solve: 1. Initialize left and right pointers at 0. 2. Iterate right pointer and update state. 3. While condition is invalid, update state removing left element and increment left. 4. Update global answer (max/min length).

C++ Template

int variableSlidingWindow(vector<int>& arr, int target) {
    int left = 0, right = 0, maxLen = 0, currentSum = 0;
    while(right < arr.size()) {
        currentSum += arr[right]; // Add element at right pointer
        
        while(currentSum > target) { // Condition violated, shrink from left
            currentSum -= arr[left];
            left++; 
        }
        maxLen = max(maxLen, right - left + 1); // Update best valid answer
        right++; // Expand window
    }
    return maxLen;
}

2. Two Pointers

▼

Deep Theory & Concept

Similar to Sliding Window but pointers generally move towards each other from opposite ends, or move at different paces. Highly dependent on the data being sorted. Because they process elements linearly and converge, time complexity is O(N).

Common Variants

1. Opposite Ends (Start at 0 and N-1, move inwards. E.g., Pair Sum, Palindrome).
2. Same Direction (One pointer slow, one fast. E.g., removing duplicates in place).

How to Identify: Sorted arrays. Problems asking for pairs, triplets, or elements satisfying an equation (like A+B=C). In-place array manipulation without extra space.

Approach to Solve: For Opposite Ends: 1. left = 0, right = n-1. 2. Calculate current state (e.g., sum = arr[left]+arr[right]). 3. If sum < target, we need a bigger number, so left++. 4. If sum > target, we need a smaller number, so right--.

C++ Template

vector<int> twoSumSorted(vector<int>& arr, int target) {
    int left = 0, right = arr.size() - 1;
    while(left < right) {
        int sum = arr[left] + arr[right];
        if(sum == target) return {left, right};
        if(sum < target) left++; // Need larger sum
        else right--; // Need smaller sum
    }
    return {-1, -1};
}

3. Fast & Slow Pointers (Floyd's Cycle)

▼

Deep Theory & Concept

In a linked list (or implicit graph), if a cycle exists, a fast pointer (moving 2 steps) will eventually lap and meet the slow pointer (moving 1 step). Furthermore, the meeting point mathematically helps find the exact start of the cycle.

Common Variants

1. Cycle Detection (Do they meet?).
2. Middle element finding (When fast reaches end, slow is at exact middle).
3. Finding cycle starting node.

How to Identify: Linked Lists. 'Find middle', 'detect cycle/loop', 'find duplicate number' (where array values are indices).

Approach to Solve: 1. Initialize slow = head, fast = head. 2. Loop while fast and fast->next are not null. 3. slow = slow->next, fast = fast->next->next. 4. If slow == fast, cycle exists.

C++ Template

ListNode* detectCycleStart(ListNode *head) {
    ListNode *slow = head, *fast = head;
    bool hasCycle = false;
    
    while(fast != NULL && fast->next != NULL) {
        slow = slow->next;
        fast = fast->next->next;
        if(slow == fast) {
            hasCycle = true;
            break;
        }
    }
    if(!hasCycle) return NULL;
    
    // To find cycle start, move slow to head, keep fast at meeting point.
    // Move both 1 step at a time. They will meet at cycle start.
    slow = head;
    while(slow != fast) {
        slow = slow->next;
        fast = fast->next;
    }
    return slow;
}

4. Merge Intervals

▼

Deep Theory & Concept

When dealing with intervals `[start, end]`, if you sort the array by `start` time, any overlapping intervals will be strictly adjacent in the sorted array. This allows you to process merges in a single linear pass `O(N)` after an `O(N log N)` sort.

Common Variants

1. Merge overlapping.
2. Insert new interval.
3. Check if any overlap exists (Meeting Rooms).
4. Find free time (Intersection).

How to Identify: Arrays of ranges `[a, b]`. Keywords: 'overlapping', 'intervals', 'scheduling', 'meetings'.

Approach to Solve: 1. Sort intervals by start time. 2. Create a result list and add the first interval. 3. Iterate: if current interval's start <= result's last interval's end, merge them (update end = max(end1, end2)). 4. Else, no overlap, push current to result.

C++ Template

vector<vector<int>> merge(vector<vector<int>>& intervals) {
    if(intervals.empty()) return {};
    // 1. Sort by start time
    sort(intervals.begin(), intervals.end());
    
    vector<vector<int>> res;
    res.push_back(intervals[0]);
    
    for(int i = 1; i < intervals.size(); i++) {
        // Overlap condition
        if(res.back()[1] >= intervals[i][0]) {
            res.back()[1] = max(res.back()[1], intervals[i][1]);
        } else {
            res.push_back(intervals[i]);
        }
    }
    return res;
}

5. Cyclic Sort

▼

Deep Theory & Concept

If you have an array of size N, and elements are strictly in the range 1 to N, every element implicitly belongs at index `value - 1`. By iterating and swapping elements to their correct index, we sort the array in O(N) without O(N log N) comparison sorts.

Common Variants

1. Find missing number.
2. Find duplicate number.
3. Find all duplicates/missing (requires second pass).

How to Identify: Numbers are in a bounded range `1 to N` or `0 to N`. Asks for missing, duplicate, or smallest missing positive integer.

Approach to Solve: 1. Iterate index i from 0 to N. 2. If arr[i] > 0 and arr[i] <= N and arr[i] != arr[arr[i]-1], swap(arr[i], arr[arr[i]-1]). 3. Else i++. 4. Second pass: first index where arr[i] != i+1 gives the missing/duplicate.

C++ Template

void cyclicSortFindMissing(vector<int>& nums) {
    int i = 0;
    int n = nums.size();
    // Pass 1: Place elements at correct index
    while (i < n) {
        int correctIndex = nums[i] - 1;
        if (nums[i] > 0 && nums[i] <= n && nums[i] != nums[correctIndex]) {
            swap(nums[i], nums[correctIndex]);
        } else {
            i++;
        }
    }
    // Pass 2: Find the mismatch
    for(int i=0; i < n; i++) {
        if(nums[i] != i + 1) {
            cout << "Missing number is: " << i + 1;
            return;
        }
    }
}

6. Top 'K' Elements (Heaps)

▼

Deep Theory & Concept

Sorting takes O(N log N). However, if we only need top K, we can use a Priority Queue (Heap). A Heap of size K takes O(log K) for insertions. Processing N elements takes O(N log K), which is much faster than N log N when K is small.

Common Variants

1. Top K Largest (Use Min-Heap of size K).
2. Top K Smallest (Use Max-Heap of size K).
3. Kth element in a stream.

How to Identify: Problem explicitly asks for 'top k', 'kth largest/smallest', 'most frequent'. Data might be streaming (unknown N).

Approach to Solve: For Top K Largest: 1. Create Min-Heap. 2. Insert first K elements. 3. For remaining elements, if element > heap.top(), pop top and push new element. 4. Heap retains the K largest seen so far.

C++ Template

vector<int> topKFrequent(vector<int>& nums, int k) {
    unordered_map<int, int> freq;
    for(int n : nums) freq[n]++;
    
    // Min heap: stores pair(frequency, number). Sorts by frequency.
    priority_queue<pair<int,int>, vector<pair<int,int>>, greater<pair<int,int>>> minHeap;
    
    for(auto& p : freq) {
        minHeap.push({p.second, p.first});
        if(minHeap.size() > k) {
            minHeap.pop(); // Remove lowest frequency
        }
    }
    
    vector<int> res;
    while(!minHeap.empty()) {
        res.push_back(minHeap.top().second);
        minHeap.pop();
    }
    return res;
}

7. Modified Binary Search

▼

Deep Theory & Concept

Standard binary search requires strict sorting. In a rotated sorted array (e.g., 4,5,6,1,2,3), at least one half of the array divided by `mid` will ALWAYS be strictly sorted. We use this sorted half to decide if our target lies within it.

Common Variants

1. Order agnostic (don't know if ascending/descending).
2. Rotated arrays.
3. Finding floor/ceiling (lower_bound, upper_bound).
4. Binary Search on Answer (Optimization problems).

How to Identify: O(log N) time required. Array is sorted or rotated. 'Find minimum in rotated array'. Optimization: 'Find the minimum capacity required to ship packages in D days'.

Approach to Solve: Rotated Array: 1. calc mid. 2. check if left to mid is sorted (`arr[left] <= arr[mid]`). 3. If it is, check if target is in range [left, mid]. If yes, right = mid-1. Else left = mid+1. 4. If right half is sorted, do inverse.

C++ Template

int searchRotated(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    while(left <= right) {
        int mid = left + (right - left) / 2;
        if(nums[mid] == target) return mid;
        
        // Is Left Half Sorted?
        if(nums[left] <= nums[mid]) { 
            // Is target inside left half?
            if(target >= nums[left] && target < nums[mid]) right = mid - 1;
            else left = mid + 1; // Go right
        } 
        // Right Half must be sorted
        else { 
            // Is target inside right half?
            if(target > nums[mid] && target <= nums[right]) left = mid + 1;
            else right = mid - 1; // Go left
        }
    }
    return -1;
}

8. Tree Depth First Search (DFS)

▼

Deep Theory & Concept

DFS uses a Stack (implicitly via function call stack in recursion). It goes deep into leaves. There are 3 ways to process a node relative to its children: Pre-order (Node, L, R), In-order (L, Node, R - gives sorted order for BST), Post-order (L, R, Node - useful when deleting or aggregating child data).

Common Variants

1. Path finding (root to leaf).
2. Subtree property aggregation (e.g. max depth, diameter).
3. Binary Search Tree validation.

How to Identify: Tree problems asking for paths, finding a specific node deeply, checking properties from root to leaf, finding lowest common ancestor.

Approach to Solve: Recursive implementation: 1. Base case (root == null). 2. Do some logic (Pre-order). 3. Recursively call DFS(left). 4. Do some logic (In-order). 5. Recursively call DFS(right). 6. Return aggregated result to parent.

C++ Template

int maxDepth(TreeNode* root) {
    if (root == NULL) return 0; // Base case
    
    int leftDepth = maxDepth(root->left);
    int rightDepth = maxDepth(root->right);
    
    // Process after children are processed (Post-order)
    return max(leftDepth, rightDepth) + 1;
}

// Check Path Sum
bool hasPathSum(TreeNode* root, int targetSum) {
    if (!root) return false;
    if (!root->left && !root->right) return targetSum == root->val;
    return hasPathSum(root->left, targetSum - root->val) || 
           hasPathSum(root->right, targetSum - root->val);
}

9. Tree Breadth First Search (BFS)

▼

Deep Theory & Concept

BFS uses a Queue. It processes all nodes at depth D before moving to depth D+1. Essential for finding the 'shortest path' in unweighted graphs or dealing with level-specific tree logic.

Common Variants

1. Level Order Traversal.
2. Zigzag traversal.
3. Shortest Path in Matrix (using BFS on grids).

How to Identify: Keywords: 'level order', 'shortest path', 'closest', 'minimum steps', 'distance from root'.

Approach to Solve: 1. Push root to Queue. 2. While Queue not empty: get queue size `s` (number of nodes in current level). 3. Loop `s` times: pop node, process it, push its left and right children if they exist. 4. Move to next level.

C++ Template

vector<vector<int>> levelOrder(TreeNode* root) {
    if (!root) return {};
    vector<vector<int>> res;
    queue<TreeNode*> q;
    q.push(root);
    
    while (!q.empty()) {
        int levelSize = q.size();
        vector<int> currentLevel;
        
        for (int i = 0; i < levelSize; ++i) {
            TreeNode* curr = q.front(); q.pop();
            currentLevel.push_back(curr->val);
            
            if (curr->left) q.push(curr->left);
            if (curr->right) q.push(curr->right);
        }
        res.push_back(currentLevel);
    }
    return res;
}

10. Backtracking (Subsets & Combinations)

▼

Deep Theory & Concept

It's an optimized brute force. Time complexity is usually exponential O(2^N) or factorial O(N!). We build a solution incrementally, and if we hit constraints, we undo the last step and try the next branch.

Common Variants

1. Subsets (include/exclude element).
2. Permutations (ordering matters, N!).
3. Combinations (target sums).
4. Constraint satisfaction (Sudoku, N-Queens).

How to Identify: Asking for 'ALL possible', 'combinations', 'permutations', 'subsets'. Input size N is very small (N <= 20).

Approach to Solve: Recursive template: `void backtrack(state, choices) {` ` if(isGoal(state)) add to result; return;` ` for(choice in choices) {` ` if(isValid(choice)) {` ` makeChoice();` ` backtrack(newState, remainingChoices);` ` undoChoice(); // BACKTRACK` ` }` ` }` `}`

C++ Template

void solveSubsets(vector<int>& nums, int start, vector<int>& curr, vector<vector<int>>& res) {
    // Add current subset to result
    res.push_back(curr);
    
    for (int i = start; i < nums.size(); i++) {
        // Choice: Include element
        curr.push_back(nums[i]);
        
        // Explore deeper
        solveSubsets(nums, i + 1, curr, res);
        
        // Backtrack: Undo choice to explore excluding element
        curr.pop_back();
    }
}
vector<vector<int>> subsets(vector<int>& nums) {
    vector<vector<int>> res;
    vector<int> curr;
    solveSubsets(nums, 0, curr, res);
    return res;
}

11. Dynamic Programming (0/1 Knapsack)

▼

Deep Theory & Concept

Knapsack is a classic 2D DP pattern. You have limited capacity and items with weights/values. For each item, you have a binary choice: take it or leave it. We memoize the state `[index][current_capacity]` to drop complexity from `O(2^N)` to `O(N * W)`.

Common Variants

1. 0/1 Knapsack (item used once).
2. Unbounded Knapsack (item used unlimited times, e.g. Coin Change).
3. Subset Sum (bool target sum).

How to Identify: Optimization (max/min) or counting ways. 'Given constraints/capacity'. 'Select elements to maximize value'.

Approach to Solve: 1. Define State: dp[i][w] = max value using first i items and capacity w. 2. Base case: capacity 0 = 0 value. 3. Transitions: Exclude: dp[i-1][w] Include: val[i-1] + dp[i-1][w - weight[i-1]] Take max of Include/Exclude.

C++ Template

int knapsack(vector<int>& wt, vector<int>& val, int W, int n) {
    // dp[items][capacity]
    vector<vector<int>> dp(n + 1, vector<int>(W + 1, 0));
    
    for(int i = 1; i <= n; i++) {
        for(int w = 1; w <= W; w++) {
            if(wt[i-1] <= w) {
                // Max of taking the item vs not taking it
                dp[i][w] = max(dp[i-1][w], val[i-1] + dp[i-1][w - wt[i-1]]);
            } else {
                // Item too heavy, must skip
                dp[i][w] = dp[i-1][w];
            }
        }
    }
    return dp[n][W];
}

12. Monotonic Stack

▼

Deep Theory & Concept

If we want to find the 'Next Greater Element' for every element in an array, nested loops take O(N^2). A monotonic stack processes elements and keeps a history of unresolved elements (elements waiting for their 'next greater'). When a larger element arrives, it resolves the stack items. Time: O(N) since every element is pushed/popped exactly once.

Common Variants

1. Next/Previous Greater/Smaller.
2. Stock Span.
3. Histogram Rectangles.

How to Identify: Problems asking for the 'next greater', 'next smaller' or involving finding spans/rectangles in arrays.

Approach to Solve: 1. Initialize empty stack (store indices, not values). 2. Iterate through array. 3. While stack not empty AND current element > arr[stack.top()]: Current element is the 'Next Greater' for the element at stack.top(). Pop it. 4. Push current index to stack.

C++ Template

vector<int> nextGreaterElement(vector<int>& nums) {
    vector<int> res(nums.size(), -1); // Default to -1
    stack<int> st; // Stores INDICES
    
    for(int i = 0; i < nums.size(); i++) {
        // While current is greater than the element at stack top's index
        while(!st.empty() && nums[i] > nums[st.top()]) {
            res[st.top()] = nums[i]; // Resolve
            st.pop();
        }
        st.push(i); // Push unresolved index
    }
    return res;
}

13. Topological Sort (Graph)

▼

Deep Theory & Concept

Essential for dependency resolution. If Course B requires Course A, A must come before B. We use Kahn's Algorithm (BFS based). We track 'in-degrees' (number of incoming edges). Nodes with 0 in-degrees have no prerequisites and can be processed immediately.

Common Variants

1. Can finish all courses? (Cycle detection in Directed Graph).
2. Return valid ordering.
3. Alien Dictionary (finding alphabet order).

How to Identify: Dependencies, scheduling tasks, 'prerequisites', 'can finish'. Graph is Directed.

Approach to Solve: 1. Build adjacency list and compute in-degree array. 2. Push all nodes with in-degree 0 to a Queue. 3. Pop node, add to topo sort list. 4. For each neighbor, reduce its in-degree by 1. If it becomes 0, push to Queue. 5. If sorted list size != total nodes, there's a cycle.

C++ Template

vector<int> topoSort(int V, vector<int> adj[]) {
    vector<int> inDegree(V, 0);
    // 1. Calculate in-degrees
    for(int i=0; i<V; i++) {
        for(int neighbor : adj[i]) inDegree[neighbor]++;
    }
    
    queue<int> q;
    // 2. Push independent nodes
    for(int i=0; i<V; i++) {
        if(inDegree[i] == 0) q.push(i);
    }
    
    vector<int> topo;
    while(!q.empty()) {
        int node = q.front(); q.pop();
        topo.push_back(node);
        
        // 3. Resolve dependencies
        for(int neighbor : adj[node]) {
            inDegree[neighbor]--;
            if(inDegree[neighbor] == 0) q.push(neighbor);
        }
    }
    // Check for cycle: if topo.size() != V, DAG has a cycle.
    return topo;
}

14. Disjoint Set / Union Find

▼

Deep Theory & Concept

Provides near O(1) time complexity for two operations: `find(A)` (which set does A belong to?) and `union(A, B)` (merge sets containing A and B). It uses an array `parent` where `parent[i]` points to the parent of `i`. Two optimizations make it lightning fast: Path Compression and Union by Rank.

Common Variants

1. Cycle detection in Undirected Graph.
2. Number of connected components.
3. Minimum Spanning Tree (Kruskal's).

How to Identify: Graph problems asking to connect components, check if nodes are in same group, dynamic connectivity.

Approach to Solve: 1. Initialize parent[i] = i. 2. `find(i)`: recursively find root. Path Compression: `parent[i] = find(parent[i])`. 3. `union(i, j)`: Find roots of i and j. If different, set one root's parent to the other.

C++ Template

class DSU {
    vector<int> parent;
    vector<int> rank;
public:
    DSU(int n) {
        parent.resize(n); rank.resize(n, 0);
        for(int i=0; i<n; i++) parent[i] = i;
    }
    
    // Find with Path Compression
    int find(int i) {
        if(parent[i] == i) return i;
        return parent[i] = find(parent[i]); 
    }
    
    // Union by Rank
    bool unite(int i, int j) {
        int rootI = find(i);
        int rootJ = find(j);
        if(rootI == rootJ) return false; // Already connected, cycle detected!
        
        if(rank[rootI] < rank[rootJ]) parent[rootI] = rootJ;
        else if(rank[rootI] > rank[rootJ]) parent[rootJ] = rootI;
        else {
            parent[rootJ] = rootI;
            rank[rootI]++;
        }
        return true;
    }
};

15. Prefix Sum

▼

Deep Theory & Concept

If you need the sum of a subarray from index L to R frequently, calculating it takes O(N) every time. By building a prefix array where `P[i] = P[i-1] + arr[i]`, the sum of `[L, R]` becomes `P[R] - P[L-1]`, solving it in O(1).

Common Variants

1. 1D Prefix Sum.
2. 2D Prefix Sum (Matrix sums).
3. Hash Map + Prefix Sum (Finding subarrays summing to K).

How to Identify: Querying sums of subarrays. Keywords: 'subarray sum', 'divisible by k', 'contiguous array'.

Approach to Solve: Hash Map + Prefix (Subarray Sum == K): 1. Maintain running `sum`. 2. Store frequencies of prefix sums in map. 3. If `sum - K` exists in map, add its frequency to result count. 4. Add current `sum` to map.

C++ Template

int subarraySum(vector<int>& nums, int k) {
    unordered_map<int, int> prefixMap;
    prefixMap[0] = 1; // Base case: sum 0 exists 1 time
    
    int currentSum = 0, count = 0;
    
    for(int num : nums) {
        currentSum += num;
        
        // If (currentSum - k) exists, we found a valid subarray ending here
        if(prefixMap.count(currentSum - k)) {
            count += prefixMap[currentSum - k];
        }
        
        prefixMap[currentSum]++;
    }
    return count;
}

16. Trie (Prefix Tree)

▼

Deep Theory & Concept

Instead of storing strings in a Hash Set (which takes O(L) time but hashes the entire string and handles collisions), a Trie stores strings character by character. A node has an array of 26 pointers (for a-z) to next nodes. It provides O(L) exact and prefix searches.

Common Variants

1. Standard Trie (Insert, Search, StartsWith).
2. Bit Trie (Storing binary representations for Maximum XOR problems).

How to Identify: Dictionaries, autocomplete, spell checkers, prefix matching, XOR problems.

Approach to Solve: 1. Create TrieNode with `TrieNode* children[26]` and `bool isEndOfWord`. 2. Insert: iterate characters, if child null, create new node, move pointer. Set `isEnd` at final node. 3. Search: similar traversal, return true if reached end and `isEnd` is true.

C++ Template

class TrieNode {
public:
    TrieNode* children[26] = {NULL};
    bool isEndOfWord = false;
};

class Trie {
    TrieNode* root;
public:
    Trie() { root = new TrieNode(); }
    
    void insert(string word) {
        TrieNode* node = root;
        for(char c : word) {
            int idx = c - 'a';
            if(!node->children[idx]) node->children[idx] = new TrieNode();
            node = node->children[idx];
        }
        node->isEndOfWord = true;
    }
    
    bool search(string word) {
        TrieNode* node = root;
        for(char c : word) {
            int idx = c - 'a';
            if(!node->children[idx]) return false;
            node = node->children[idx];
        }
        return node->isEndOfWord;
    }
};

17. In-place Reversal of a LinkedList

▼

Deep Theory & Concept

By manipulating the `next` pointers of nodes, we can reverse the list traversing it only once O(N) and using O(1) space.

Common Variants

1. Reverse full list.
2. Reverse sub-list (from position M to N).
3. Reverse in K-groups.

How to Identify: Linked List problems asking for reversal, checking palindrome, or reordering.

Approach to Solve: 1. Need 3 pointers: `prev` (NULL), `curr` (head), `next` (temp). 2. While `curr` != NULL: `next = curr->next;` (save next node) `curr->next = prev;` (reverse link) `prev = curr;` (move prev forward) `curr = next;` (move curr forward) 3. Return `prev` (new head).

C++ Template

ListNode* reverseList(ListNode* head) {
    ListNode* prev = NULL;
    ListNode* curr = head;
    
    while (curr != NULL) {
        ListNode* nextTemp = curr->next; // Store next
        curr->next = prev;               // Reverse link
        prev = curr;                     // Advance prev
        curr = nextTemp;                 // Advance curr
    }
    return prev; // Prev is the new head
}

18. K-way Merge (Heaps)

▼

Deep Theory & Concept

If you merge K lists linearly, it takes O(N*K). Using a Min-Heap of size K, we insert the first element of each list. We extract the min element, and insert the NEXT element from the same list that the min element came from. Time complexity becomes O(N log K) where N is total elements.

Common Variants

1. Merge K Linked Lists.
2. Kth Smallest in M Sorted Arrays.
3. Smallest Range covering K lists.

How to Identify: Input is K sorted arrays, lists, or matrices. Need to merge them or find median/kth element across them.

Approach to Solve: 1. Create Min-Heap of custom objects (storing value, array_index, element_index). 2. Push first element of all K arrays. 3. Pop min, add to result. 4. If the array from which min came has more elements, push its next element. 5. Repeat until heap is empty.

C++ Template

ListNode* mergeKLists(vector<ListNode*>& lists) {
    // Custom comparator for Min Heap of ListNodes
    auto comp = [](ListNode* a, ListNode* b) { return a->val > b->val; };
    priority_queue<ListNode*, vector<ListNode*>, decltype(comp)> pq(comp);
    
    // Push the head of all K lists
    for(auto node : lists) {
        if(node) pq.push(node);
    }
    
    ListNode dummy(0);
    ListNode* tail = &dummy;
    
    while(!pq.empty()) {
        ListNode* minNode = pq.top(); pq.pop();
        tail->next = minNode;
        tail = tail->next;
        
        // Push next node from the same list
        if(minNode->next) pq.push(minNode->next);
    }
    return dummy.next;
}

19. Two Heaps

▼

Deep Theory & Concept

By splitting data into a lower half (Max Heap) and upper half (Min Heap), the median is always at the top of the heaps. Insertion is O(log N) and finding median is O(1).

Common Variants

1. Data Stream Median.
2. Sliding Window Median.

How to Identify: Dynamic calculation of Medians. 'Stream of numbers'.

Approach to Solve: 1. Keep maxHeap for lower half, minHeap for upper half. 2. Insert: If num <= maxHeap.top(), push to maxHeap. Else push to minHeap. 3. Balance: Sizes must differ by at most 1. If maxHeap size > minHeap size + 1, pop from max and push to min. 4. Median: If odd, top of larger heap. If even, average of tops.

C++ Template

class MedianFinder {
    priority_queue<int> maxHeap; // Lower half
    priority_queue<int, vector<int>, greater<int>> minHeap; // Upper half
public:
    void addNum(int num) {
        if (maxHeap.empty() || num <= maxHeap.top()) maxHeap.push(num);
        else minHeap.push(num);
        
        // Balance
        if (maxHeap.size() > minHeap.size() + 1) {
            minHeap.push(maxHeap.top()); maxHeap.pop();
        } else if (minHeap.size() > maxHeap.size()) {
            maxHeap.push(minHeap.top()); minHeap.pop();
        }
    }
    
    double findMedian() {
        if (maxHeap.size() == minHeap.size())
            return (maxHeap.top() + minHeap.top()) / 2.0;
        return maxHeap.top(); // maxHeap is designed to hold the extra element
    }
};

20. Bit Manipulation

▼

Deep Theory & Concept

Bit manipulation is extremely fast O(1) space and time. Key facts: `X ^ X = 0`, `X ^ 0 = X`, `X & (X-1)` removes the lowest set bit. Useful in subset generation and finding missing items.

Common Variants

1. XOR properties (Single Number).
2. Bit Masking (representing sets as integers).
3. Counting set bits.

How to Identify: Constraints are small (N < 32), checking parity, 'find element that appears once'.

Approach to Solve: Single Number logic: Since A ^ A = 0, XORing all elements in an array where every element appears twice except one will leave the unique element.

C++ Template

// 1. Find Single Number
int singleNumber(vector<int>& nums) {
    int ans = 0;
    for(int num : nums) ans ^= num;
    return ans;
}

// 2. Count Set Bits (Kernighan's Algorithm)
int countSetBits(int n) {
    int count = 0;
    while(n) {
        n = n & (n - 1); // clears lowest set bit
        count++;
    }
    return count;
}

Time Complexity Mastery

1. Deep Theory of Time Complexity

Time complexity isn't about seconds; it's about the rate of growth of operations relative to the input size `N`. We focus on the worst-case scenario (Big-O).

  • O(1) Constant: Time does not change with N. (e.g., Array lookup `arr[5]`, Hash Map lookup, Math formulas like `N*(N+1)/2`).
  • O(log N) Logarithmic: Data is halved at every step. (e.g., Binary Search, traversing a balanced BST). If you see a sorted array and require an algorithm faster than O(N), it's O(log N).
  • O(N) Linear: Visiting every element a fixed number of times. (e.g., Iterating an array, Sliding Window).
  • O(N log N) Log-Linear: Doing an O(log N) operation N times. (e.g., Sorting an array, pushing N elements into a Heap/Priority Queue).
  • O(N^2) Quadratic: Comparing every element with every other element. (e.g., Nested loops, Bubble sort). Usually acceptable only if `N < 1000`.
  • O(2^N) Exponential: Branching into two paths at every step. (e.g., finding all subsets, basic recursive Fibonacci).
  • O(N!) Factorial: Finding all permutations of a string.

2. How to Decide Required Complexity from Constraints

The "10^8 Rule"

Modern competitive programming servers (like LeetCode, Codeforces, HackerRank) can process roughly 10^8 operations per second in C++. If your algorithm's worst-case operations exceed 10^8, you will get a Time Limit Exceeded (TLE).

Before writing a single line of code, look at the constraints provided in the problem description to reverse-engineer the required time complexity:

Constraint on N Target Time Complexity Algorithms / Patterns to apply
N ≤ 10...12O(N!) or O(2^N)Backtracking (Permutations, N-Queens)
N ≤ 20...25O(2^N) or O(N · 2^N)Backtracking (Subsets), Bitmask DP
N ≤ 100O(N^3) or O(N^4)Floyd-Warshall, 3D/4D Dynamic Programming
N ≤ 10^3...10^4O(N^2)Nested Loops, 2D DP (0/1 Knapsack)
N ≤ 10^5...10^6O(N log N) or O(N)Sorting, Binary Search, Heaps, Sliding Window, Two Pointers
N ≤ 10^7...10^8O(N)Strictly linear. Prefix Arrays, Hash Maps. Avoid Sorting.
N ≥ 10^9O(log N) or O(1)Binary Search on Answer, Mathematical formulas

3. How to Manage and Optimize Complexity

If you write a brute force `O(N^2)` solution but the constraint is `10^5`, you must optimize. Here is the standard playbook to manage time complexity:

  1. Space-Time Tradeoff: Can you use extra memory to save time? Use a Hash Map to store previously seen elements to turn an `O(N^2)` search into `O(N)`. (Pattern: Two Sum).
  2. Precomputation: Do you calculate the same sum multiple times? Compute a Prefix Sum array once in `O(N)` so future queries take `O(1)`.
  3. Memoization (DP): Is your recursion calculating the same subproblems repeatedly? Store the results in a matrix/array to drop from `O(2^N)` down to `O(N)` or `O(N^2)`.
  4. Sort Data First: Does order matter? If not, sorting takes `O(N log N)` but allows you to use Two Pointers `O(N)` or Binary Search `O(log N)` subsequently.
  5. Lazy Evaluation / Monotonic Stack: Don't process elements until you have to. If looking for the "Next Greater Element", use a stack to defer processing, achieving `O(N)` instead of nested loops.
